# Agentic Control System – Dokumentenpaket

Dieses Paket bündelt die Architektur- und Leitplankendokumente für die Implementierung
eines persönlichen Multi-Projekt-Agentic-Control-Systems mit Codex CLI und Claude Code.

## Empfohlene Lesereihenfolge
1. `01-solution-brief.md`
2. `02-reference-architecture.md`
3. `03-architecture-guardrails.md`
4. `04-bounded-contexts.md`
5. `05-state-and-lifecycle-model.md`
6. `06-trust-failure-and-escalation.md`
7. `07-implementation-handoff.md`
8. `08-architecture-decisions.md`
9. `09-canonical-domain-model.md`
10. `10-operating-model.md`
11. `11-glossary.md`

## Ziel des Pakets
Die Dokumente sollen:
- das Zielsystem fachlich und architektonisch fixieren,
- spätere Erosion durch pragmatische Abkürzungen verhindern,
- und Codex CLI / Claude Code mit sauber getrennten Markdown-Artefakten versorgen.
